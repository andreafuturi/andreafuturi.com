## Cursor Cloud specific instructions

### Project overview

Personal portfolio site for Andrea Futuri (`andreafuturi.com`). Deno serves the static HTML/JS/SVG files; Vite + Preact used for client-side development tooling.

### Runtimes required

- **Deno** — serves the HTTP server (`server/main.jsx`), listens on port 8000
- **Node.js + npm** — installs Vite/Preact dev dependencies

### Running the dev environment

Standard commands are in `package.json`:

- `npm run dev-server` — Deno server with `--watch` + `--dev` flag (port 8000)
- `npm run dev-client` — Vite client dev server with HMR
- `npm start` — runs both concurrently

### Gotchas

- The `--unstable` flag in the Deno commands produces a warning on Deno 2.x ("removed in Deno 2.0, use granular `--unstable-*` flags"). This is cosmetic and does not affect functionality.
- The client source code that Vite would build is **not** in the repo — only the pre-built bundle (`client/portfolio/andreafuturi.com/index.js`) is committed. Running `npm run dev-client` (Vite) requires a `client/index.html` which is generated by `npm run create-index`. The Deno server alone is sufficient to serve and test the portfolio.
- No database, no secrets, no `.env` files required.
- No linter or test framework is configured in this project.
